# 【Code with SOLO】用 SOLO 从零搭建 AI 职场行动教练——全链路求职助手

## 1. 摘要

使用 TRAE SOLO 从零搭建了一个名为 **CareerPilot（AI职场行动教练）** 的全链路求职辅助 Web 应用。涵盖岗位解读、行动规划、简历优化、面试练习四大核心模块，并创新性地提供"求职一条龙"一键式服务。整个项目从需求分析到部署上线，全程使用 TRAE SOLO 完成，无需后端，纯前端实现。

## 2. 背景

我是一名在校大学生，身边很多同学在求职时面临共同的困境：不知道岗位真实工作内容是什么、不知道该补什么能力、简历不知道怎么写、面试不知道怎么准备。市面上的求职工具要么功能单一，要么收费昂贵。

我希望用 AI 技术打造一个**免费、全链路、开箱即用**的求职助手，帮助大学生从"迷茫"到"行动"。

## 3. 实践过程

### 3.1 任务拆解

我将整个项目拆解为以下阶段：

1. **需求分析与技术选型** → 确定使用 React + TypeScript + TailwindCSS + Vite
2. **核心架构搭建** → 路由、布局、主题系统、AI 服务层
3. **四大功能模块开发** → 岗位翻译器、行动规划师、简历优化、面试练习
4. **求职一条龙整合** → 将四个模块串联为一键式服务
5. **UI/UX 精细化** → 白天/夜间主题、响应式布局、SVG 图标、交互优化
6. **部署与发布** → 推送 GitHub，编写详细 README

### 3.2 使用 SOLO 的关键过程

**技术选型阶段：** 我直接向 SOLO 描述了项目需求，SOLO 帮我确定了技术栈（React 19 + Vite 8 + TailwindCSS 4），并解释了为什么选择这些技术。

**AI 服务层设计：** 我让 SOLO 设计 AI 调用层，它帮我实现了基于 `fetch` + `ReadableStream` 的 SSE 流式输出方案，支持打字机效果实时展示 AI 回复。关键 Prompt 如：

> "帮我设计一个流式调用硅基流动 API 的服务层，支持 SSE 流式读取，使用 fetch + ReadableStream，要有错误处理和超时机制"

**Prompt 工程：** 每个功能模块都有精心设计的 System Prompt。例如岗位翻译器的 Prompt：

> "你是一位资深职场导师。请分析以下JD，输出：真实工作场景（日常做什么）、硬技能要求、软技能要求、不适合人群。用 Markdown 格式输出，语言简洁有力。"

**求职一条龙功能：** 这是最复杂的部分。SOLO 帮我设计了"信息录入 → 4步顺序执行 → 结果总览"的完整流程，使用 `useRef` 解决了异步流式调用中的闭包陷阱问题。

**UI 精细化：** 在开发过程中，我持续让 SOLO 优化 UI 细节，包括：
- 白天蓝天白云、夜间星空流星的动态背景
- 玻璃态（Glassmorphism）卡片设计
- TailwindCSS v4 的 `@variant dark` 配置（解决了暗色模式不生效的问题）
- SVG 图标替换 emoji（跨设备显示一致性）
- 历史记录的删除确认对话框

### 3.3 踩过的坑

1. **TailwindCSS v4 暗色模式不生效**：v4 默认使用 `prefers-color-scheme` 而非 `.dark` 类选择器，需要手动添加 `@variant dark (&:where(.dark, .dark *))` 配置
2. **CSS @import 顺序错误**：`@import url()` 必须放在 `@variant` 之前，否则 PostCSS 会报错导致样式失效
3. **useStreamChat 闭包陷阱**：在一条龙模式中，`useCallback` 的闭包引用了旧的 state，需要用 `useRef` 来跟踪最新值
4. **密码框粘贴限制**：`type="password"` 输入框在某些浏览器中无法粘贴，需要手动处理 `onPaste` 事件并调用 `e.preventDefault()`

## 4. 成果展示

### 项目地址

### 核心功能

| 功能 | 说明 |
|------|------|
| 🚀 求职一条龙 | 输入一次信息，AI 依次完成 4 个步骤，生成完整求职报告 |
| 📋 岗位翻译器 | 粘贴 JD，AI 解读真实工作内容、技能要求、不适合人群 |
| 📝 行动规划师 | 输入背景信息，AI 生成能力诊断 + 4 阶段行动清单 |
| 📄 简历优化 | 粘贴简历片段，AI 诊断问题并给出 STAR 法则改写建议 |
| 🎤 面试练习 | 模拟真实面试，AI 评分并给出改进建议，支持多轮追问 |
| 🤖 多模型支持 | 支持 Qwen、DeepSeek、Kimi 等多种 AI 模型自由切换 |
| 🌙 主题切换 | 白天蓝天白云 / 夜间星空流星，一键切换 |

### 技术亮点
- **纯前端实现**：无需后端，使用 localStorage 存储数据
- **流式输出**：基于 SSE 的打字机效果，实时展示 AI 回复
- **响应式设计**：移动端底部导航 + 桌面端侧边栏，完美适配
- **多模型支持**：在设置中自由切换 AI 模型，带连接测试功能
- **6 个页面、15+ 组件**：完整的 Web 应用架构

## 5. 效果与总结

### 提效数据
- 整个项目从零到部署上线，全程使用 TRAE SOLO 完成
- 涵盖 6 个页面、15+ 组件、4 套 AI Prompt，代码量约 8000+ 行
- 每个功能模块的开发周期约 30-60 分钟（含 UI 设计和调试）

### SOLO 在流程中的角色
- **架构师**：帮我确定技术选型和项目结构
- **编码助手**：生成组件代码、处理复杂逻辑（如流式输出、异步状态管理）
- **调试专家**：快速定位 CSS 报错、闭包陷阱、浏览器兼容性问题
- **UI 设计师**：优化视觉细节，实现玻璃态、动态背景等效果

### 可复用的经验
1. **先规划再编码**：每次开发新功能前，先用 SOLO 做需求分析和方案设计，减少返工
2. **小步快跑**：每完成一个功能就验证，而不是一次性开发完再测试
3. **善用截图反馈**：遇到 UI 问题时，截图让 SOLO 看实际效果，比文字描述更高效
4. **Prompt 是核心**：AI 功能的质量很大程度上取决于 Prompt 的设计，值得反复打磨
